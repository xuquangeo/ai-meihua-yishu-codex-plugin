# Image2 教学卡固定规范

本文件用于约束插件未来生成的 `image2 / image_gen` 教学卡版本。

## 目标

让教学卡既好看，又不改错事实。

## 前台输出硬规则

- 面向用户的最终主结果必须是 `PNG`
- 这张 `PNG` 必须由 `image2 / image_gen` 生成
- 不允许把 `SVG` 当作前台最终展示结果
- `SVG` 只能作为后台事实校准底稿

## 事实来源

教学卡里所有事实必须来自：

- `result.json`
- `upper_trigram.svg`
- `lower_trigram.svg`
- `primary_hexagram.svg`
- `changed_hexagram.svg`
- `step_by_step_report.svg`

图像模型不能自己猜卦，不能自己改字，不能自己改动爻。

## 八卦图示硬规则

八卦图示必须完全固定，不能靠模型自由生成。

- `阳` = 一整横
- `阴` = 中间断开的两短横

八卦从上到下固定是：

- 乾：阳 / 阳 / 阳
- 兑：阴 / 阳 / 阳
- 离：阳 / 阴 / 阳
- 震：阴 / 阴 / 阳
- 巽：阳 / 阳 / 阴
- 坎：阴 / 阳 / 阴
- 艮：阳 / 阴 / 阴
- 坤：阴 / 阴 / 阴

禁止出现：

- 卦名是艮，图示却画成坤
- 卦名是坤，图示却画成乾
- 动爻变化正确，但三爻名字不对
- 三爻图对了，但五行名字不对

## 固定结构

教学卡默认分四步：

1. 明确问题，给数字
2. 做计算，算本卦
3. 做计算，算变卦
4. 做解析

## 基础知识区硬规则

教学卡中必须固定保留一块“八卦图形对照区”。

这块区域的作用是：

- 让用户一眼看到八卦名称和图形的对应
- 让第二步算本卦时可以直接引用
- 让第三步算变卦时可以直接核对

这块区域至少要同时体现：

- 卦名
- 图形
- 必要时可附带五行

建议固定列出：

- 乾：阳 / 阳 / 阳
- 兑：阴 / 阳 / 阳
- 离：阳 / 阴 / 阳
- 震：阴 / 阴 / 阳
- 巽：阳 / 阳 / 阴
- 坎：阴 / 阳 / 阴
- 艮：阳 / 阴 / 阴
- 坤：阴 / 阴 / 阴

## 第三步硬规则

第三步文案必须明确表现：

- `先把两个数字相加`
- `再除以 6`
- `余数就是动爻`

例如 `73` 这个案例，必须写成：

- `7+3=10`
- `10÷6，余4`
- `动爻：第四爻`

不允许写成：

- `7÷3`
- `7 除 3`
- 任何把两个数字直接相除的错误形式

第三步在表现变卦时，要直接参照“八卦图形对照区”，不能脱离这块基础知识区自由发挥。

## 本卦组合硬规则

第二步中一旦已经算出：

- 上卦
- 下卦

那么在“本卦”那一格里，必须直接复用这两个三爻结果：

- 上卦画在上面
- 下卦画在下面

不允许出现：

- 单独展示的上卦是一个图
- 组合成本卦以后，上半部分又变成另一个图
- 单独展示的下卦是一个图
- 组合成本卦以后，下半部分又变成另一个图

也就是说，必须满足：

- `本卦上半 = 已算出的上卦图形`
- `本卦下半 = 已算出的下卦图形`

如果不一致，整张图判定为错误图。

## 第四步必须出现的内容

第四步解析必须明确画出：

- 八卦数字对应
- 八卦五行对应
- 五行生克关系
- 体用关系
- 最终结论

## 五行生克图硬规则

五行生克图必须是完整五节点，不允许缺项。

必须完整出现：

- 金
- 水
- 木
- 火
- 土

并且要同时表现两层关系：

- 相生
- 相克

### 相生固定链路

- 金生水
- 水生木
- 木生火
- 火生土
- 土生金

### 相克固定链路

- 金克木
- 木克土
- 土克水
- 水克火
- 火克金

禁止出现：

- 只有 4 个五行节点
- 少一个元素
- 生克箭头不闭环
- 只画相生，不画相克
- 只画相克，不画相生

## 八卦五行对应固定表

- 乾：金
- 兑：金
- 离：火
- 震：木
- 巽：木
- 坎：水
- 艮：土
- 坤：土

## 推荐提示词要点

Prompt 中要明确写：

- 这是一张教学卡，不是重新起卦
- 所有卦线以脚本生成结果为准
- 所有中文文字必须可读
- 要明确表现八卦与五行对应
- 要明确表现五行生克关系
- 要明确表现体用关系和最后判断
- 五行生克图必须完整出现 `金、水、木、火、土` 五个节点

## 使用原则

- 好看是加分项，正确是硬约束
- 如果 image2 版本有错字、错卦、错动爻，必须回退到确定性 SVG 版本作为标准输出
- 但即使回退到 SVG 作为事实标准，也不要把 SVG 当成前台展示稿；应继续修正 image2 PNG 版本
