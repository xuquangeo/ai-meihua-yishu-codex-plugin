# 梅花易数算本卦

这个 Skill 单独负责第二步：

- 做计算，算本卦

## 规则写死

先读取：

- `../../docs/trigram-visual-reference.md`

本卦算法固定是：

1. 第一位数字除以 `8`
2. 余数取上卦
3. 余数为 `0` 时按 `8`
4. 第二位数字除以 `8`
5. 余数取下卦
6. 余数为 `0` 时按 `8`
7. 上卦和下卦组合成本卦

## 第二步必须讲清的链路

这一层不是只报结果，要明确表现：

- `数字 -> 八卦`
- `八卦 -> 五行`

例如 `73` 这个案例，必须能展开成：

- `7 ÷ 8，余 7 -> 艮 -> 土`
- `3 ÷ 8，余 3 -> 离 -> 火`
- 组合得到：`山火贲`

## 固定映射

### 先天八卦数字对应

- 1：乾
- 2：兑
- 3：离
- 4：震
- 5：巽
- 6：坎
- 7：艮
- 8：坤

### 八卦与五行对应

- 乾：金
- 兑：金
- 离：火
- 震：木
- 巽：木
- 坎：水
- 艮：土
- 坤：土

## 图示硬规则

不要只记名字，必须把图示一起固化：

- `阳` = 一整横
- `阴` = 中间断开

八卦图示按从上到下固定是：

- 乾：阳 / 阳 / 阳
- 兑：阴 / 阳 / 阳
- 离：阳 / 阴 / 阳
- 震：阴 / 阴 / 阳
- 巽：阳 / 阳 / 阴
- 坎：阴 / 阳 / 阴
- 艮：阳 / 阴 / 阴
- 坤：阴 / 阴 / 阴

每次输出本卦时，要同时校对：

- 数字
- 卦名
- 五行
- 图示

## 输入

- 第一位数字
- 第二位数字

## 输出

- 上卦
- 下卦
- 本卦
- 数字到八卦的映射
- 八卦到五行的映射

## 本卦组合硬规则

一旦上卦和下卦算出来，后面所有地方都必须直接沿用：

- `本卦上半 = 上卦`
- `本卦下半 = 下卦`

不能在“本卦”那一格里重新自由画一个像似而非的图。

如果：

- 上卦单图是 `艮`
- 但本卦上半又画成别的三爻结构

那就是错误。

后续如果要交给出图 Skill，应该把这条事实直接写成装配锁：

- `上卦单图 = 本卦上半`
- `下卦单图 = 本卦下半`

## 与 image2 教学卡的关系

如果第二步要交给 `image2 / image_gen` 出图：

- 要明确表现上卦和下卦各自的计算
- 要明确表现 `数字 -> 八卦 -> 五行`
- 不允许只剩一个卦名，不解释过程
